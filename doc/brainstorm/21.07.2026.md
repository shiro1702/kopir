Скопировать
Да, блок с точками лучше превратить не просто в “карту с маркерами”, а в полноценный выбор точки печати. Причем одинаковая логика должна быть:

На сайте — для лендинга и SEO-страниц.
В Telegram Mini App — для оформления заказа.
В боте — как fallback, если миниаппа не открылась.
Главная идея:

Пользователь выбирает удобную точку печати по карте, списку, расстоянию, графику, цене и статусу работы.

Может нажать на точку на карте.
Открывается информационный балун/карточка.

Логика выбора точки в заказе
Лучший порядок оформления:

Вариант A — сначала файл, потом точка
1. Пользователь загружает файл
2. Бот считает страницы
3. Пользователь выбирает количество копий
4. Выбирает точку
5. Видит цену
6. Оплачивает
Плюс: точнее расчет, пользователь уже вовлечен.

Минус: точка влияет на цену, поэтому до выбора точки цена может быть “от”.

Вариант B — сначала точка, потом файл
1. Пользователь выбирает точку
2. Загружает файл
3. Выбирает количество копий
4. Видит цену
5. Оплачивает
Плюс: понятнее логистика.

Минус: если точек много, пользователь тратит время до основного действия.

Ты уже правильно описал основу:

OSM-карта;
информационные балуны;
список точек сбоку;
модальное окно;
выбор точки;
такой же функционал в миниаппе.
Я бы обязательно добавил еще:

Геолокацию пользователя.
Сортировку точек по расстоянию.
Статус “открыто/закрыто/скоро закрывается”.
Проверку доступности точки перед оплатой.
Кнопки маршрута в Яндекс.Карты/2ГИС.
Фото входа и ориентиры.
Deep link в Telegram с point_id.
Запоминание последней выбранной точки.
