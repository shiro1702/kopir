# Документация Kopir

## Быстрый старт для Cursor

1. Прочитай **[PROJECT.md](./PROJECT.md)** — что строим, стек, архитектура
2. Открой **[SPRINTS.md](./SPRINTS.md)** — что делать сейчас (Sprint 0)
3. Сверяйся с **[FEATURES.md](./FEATURES.md)** — статусы фич, не дублируй работу

## Файлы

| Файл | Назначение |
|------|------------|
| [PROJECT.md](./PROJECT.md) | Сводка проекта + технологии (контекст для AI) |
| [FEATURES.md](./FEATURES.md) | Реестр фич со статусами ⬜ 🔵 ✅ |
| [ROADMAP.md](./ROADMAP.md) | Этапы 1–4: пилот → MVP → автоматизация → масштаб |
| [SPRINTS.md](./SPRINTS.md) | Задачи по 2-недельным спринтам |
| [brainstorm/11.06.2026.md](./brainstorm/11.06.2026.md) | Исходный брейншторм (не редактировать) |

## Как обновлять статусы

В `FEATURES.md` меняй `⬜ todo` → `🔵 in_progress` → `✅ done`.

В `SPRINTS.md` отмечай задачи спринта и заполняй ретро в конце.

## Корень репозитория

`.cursorrules` — автоматически подхватывается Cursor при работе в проекте.
