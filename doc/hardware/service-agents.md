# Сервисные агенты (заправщики)

> Источники: [brainstorm/12.06.2026.md](../brainstorm/12.06.2026.md), [brainstorm/13.06.2026.md](../brainstorm/13.06.2026.md)

Система контроля наёмных заправщиков (студентов) для франчайзи — **не в Sprint 0**.

## Авторизация на точке

**Принято:** динамический PIN на OLED-экране бокса (не геолокация — плохо в подвалах вузов).

```
1. Экран показывает 4-значный код (ротация ~1 мин)
2. Агент вводит код в Telegram-бот
3. Сервер сверяет → открывает замок, стартует сессию
4. Камера фиксирует открытие
```

Альтернатива: служебный QR внутри лотка или NFC-карта.

## Цикл обслуживания

1. Автозаявка при низком остатке бумаги
2. Агент принимает заявку в боте
3. PIN → открытие замка склада
4. **Фотоотчёт:** до / после / остаток на складе
5. IoT-проверка: CUPS не должен показывать «No Paper» после «Готово»
6. Сверка: агент вводит «+500 листов» → сверка с первым заказом после ухода
7. Начисление оплаты (~150 ₽/выезд)

## Камера (~500–700 ₽)

- USB на Orange Pi
- Снимок при открытии замка + финальный кадр
- Защита от воровства бумаги со склада

## Аварийный доступ

- Студенту ключ **не выдаётся**
- При заклинивании электрозамка — владелец франшизы с механическим ключом
- Тросик от цилиндра к защёлке соленоида

## Для презентации франшизы

«Управление сетью из 10 боксов — 2 часа в неделю на просмотр фотоотчётов».

## Ячейки Student Hub (заполнение вендинга)

Отдельно от цикла «+500 листов в Tray 1» — **наполнение ячеек продаж**:

1. PIN → открытие сервисной зоны (или отдельный доступ к пустым ячейкам).
2. В админ-боте для каждой ячейки: **что положено** + **цена** (или выбор из каталога товаров).
3. Примеры: «Ячейка 1 — 50 листов, 150 ₽»; «Ячейка 2 — файлы 10 шт, 60 ₽».
4. Бот клиента обновляет меню «Мини-маркет» и остатки.
5. Сезонность: перед сессией — 100/150 листов вместо 50 (см. [student-hub-vending.md](../product/student-hub-vending.md)).

Заправщику не нужно настраивать железо — только положить товар и отметить в боте.
