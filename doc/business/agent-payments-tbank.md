## Налог (УСН 6% «Доходы»)

Доход = **только комиссия**, не весь платёж клиента.

```
Клиент 100 ₽ → комиссия 12 ₽ → налог 6% с 12 ₽ = 0.72 ₽
```

Квартальная уплата. Не «в момент перевода партнёру».

## Варианты движения денег

### Сплит (мгновенный)

- 88 ₽ → копицентр, 12 ₽ → нам при оплате
- **Минус:** сложно с плавающим % (перерасчёт в следующем месяце)

### Транзитный счёт

- Всё на наш счёт → выплаты партнёрам + **Отчёт агента** ежемесячно

## Плавающий % + агентская схема — совместимы

В оферте:

> «Размер вознаграждения по Тарифам на Сайте. Агент вправе менять ставку в зависимости от объёма заказов за отчётный период».

### Способ 1: перерасчёт на следующий месяц (рекомендуется со сплитом)

Январь — 12% в настройках банка. 1 февраля система смотрит оборот → февраль 9% или 15%.

### Способ 2: удержание в конце месяца

Все деньги к нам → отчёт → перевод партнёру с актуальным %.

### Способ 3: кэшбек

Всегда 15%, при выполнении плана — возврат 3–6% на баланс ЛК.

## Т-Банк: номинальный счёт (рекомендация)

**Не** чистое мгновенное сплитование — нужен гибкий % по обороту.

### Процесс

1. Клиент платит → **100% на номинальный счёт** (деньги юридически партнёров)
2. Бот считает оборот каждой точки
3. Раз в неделю/месяц — **реестр выплат** через API:
   - Копицентр №1: 50k оборот, 9% → 45 500 ₽ партнёру, 4 500 ₽ на наш РС
4. На наш РС приходит **только комиссия** → УСН 6% без вопросов

### Плюсы

- Любой % и пересчёт
- Партнёр может быть в любом банке (Сбер, Альфа…)
- Прозрачно для налоговой

### Запуск

1. ИП + РС в Т-Бизнес
2. Заявка «Решения для маркетплейсов / номинальный счёт»
3. API: эквайринг на номинал + скрипт реестра
4. Оферта партнёра (шаблон от банка) — галочка при регистрации в ЛК

## Старт без номинального счёта (MVP)

На этапе запуска номинальный счёт **не обязателен**.

### Свои принтеры (типичный старт)

- Номинальный счёт **не нужен** (даже в будущем)
- ИП + расчётный счёт + эквайринг (Т-Банк / ЮKassa)
- Деньги клиента → напрямую на РС; налог УСН с дохода

### Платформа / принтеры партнёров

Номинальный счёт нужен при автоматическом сплите чужих денег (115-ФЗ). **Обход на MVP:**

| Путь | Как | Плюсы / минусы |
|------|-----|----------------|
| **А. Ручной агентский расчёт** | Все деньги на РС ИП → ежемесячный отчёт агента → перевод партнёрам | Запуск за дни; 1 ч/мес при 5–10 партнёрах |
| **Б. Split платёжки** | ЮKassa / Cloudpayments / Robokassa делят платёж на своём номинале | Легально, автоматом; сложнее API |
| **В. Номинальный Т-Банк** | Полная схема маркетплейса | Когда оборот и число партнёров растут |

Рекомендация для Kopir сейчас: **путь А** + функционал из [partner-payouts.md](./partner-payouts.md).

Самозанятый — возможен для теста на 1–2 принтера (ЮKassa / Robokassa); для сети — ИП УСН 6%.
