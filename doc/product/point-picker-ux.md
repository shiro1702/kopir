## Принцип

Один и тот же **смысл выбора точки** на сайте, в Mini App и (fallback) в боте:

> Пользователь выбирает удобную точку по карте, списку, расстоянию, графику, цене и статусу работы.

Shared UI: `components/point-picker/` + composable `usePointPicker` (данные из `GET /api/points`).

## Компоненты

### 1. Маркеры на карте

Leaflet + OpenStreetMap. Опционально `Leaflet.markercluster` (Sprint 6).

| Цвет | Статус |
|------|--------|
| 🟢 Зелёный | Открыта, принимает заказы |
| ⚪ Серый | Закрыта |
| 🟡 Жёлтый | Скоро закрывается (< 30 мин) |
| 🔴 Красный | Временно недоступна (offline / no paper) |

### 2. Балун (короткий)

При клике на маркер — **не перегружать**:

```
Копицентр «Название»
ул. Примерная, 10
Открыто до 19:00 · А4 ч/б — 7 ₽/стр.
[Выбрать] [Подробнее]
```

### 3. Модалка / bottom sheet (полная карточка)

Одинаковая структура на сайте и в Mini App:

- фото входа (если есть);
- название, адрес;
- статус работы + статус приёма онлайн-заказов;
- цена А4 ч/б;
- время готовности (статическое на старте);
- как получить заказ (`pickupInstructions`);
- другие услуги (справочно, «уточняйте у оператора»);
- [Выбрать эту точку] [Построить маршрут] [Позвонить]

На мобильном — **bottom sheet**, не центрированная модалка.

### 4. Список точек

Карточка в списке:

```
Копицентр «Название»
ул. Примерная, 10
Открыто до 19:00 · 7 ₽/стр. · ≈ 850 м
Готовность: 1–3 мин
[Выбрать] [Подробнее]
```

Клик по карточке: центрировать карту, подсветить маркер, открыть bottom sheet, активная карточка в списке.

## Поведение «Выбрать точку»

| Канал | Действие |
|-------|----------|
| **Сайт** | `https://t.me/{bot}?start=point_{slug}` (и MAX-аналог) |
| **Mini App** | Сохранить `pointId` в batch → экран заказа «📍 Точка: … [Изменить]» |
| **Бот** | Inline callback / код / QR (см. bot-point-selection) |

### QR / deep link — точка уже известна

```
Вы печатаете в точке:
Копицентр «Название», ул. Примерная, 10
[Продолжить] [Выбрать другую точку]
```

Не заставлять выбирать точку повторно.

## Статусы

### График работы (текст для UI)

| Состояние | Пример текста |
|-----------|---------------|
| Открыта | «Открыто до 19:00» |
| Закрыта | «Закрыто, откроется завтра в 9:00» |
| Скоро закрывается | «Скоро закрывается — до 19:00» + предупреждение |
| Недоступна | «Временно недоступна» |

### Приём онлайн-заказов

Отдельно от «физически открыта»:

- «Принимает заказы»
- «Онлайн-заказы временно недоступны»

### Закрытая точка

MVP: **нельзя выбрать** закрытую точку. Показывать серым, без primary CTA.

## Геолокация

Кнопка «Показать точки рядом со мной» / «Определить местоположение».

После разрешения: центр карты на пользователе, расстояние в карточках, сортировка по haversine.

**Без геолокации** всё работает по списку и городу по умолчанию.

## Проверка перед оплатой

Перед Init оплаты — re-check (см. [point-availability.md](./point-availability.md)):

- точка `isActive`;
- `acceptsOnlineOrders`;
- агент online (MON-02);
- не закрылась по графику;
- цена не изменилась.

При сбое: «Точка сейчас не принимает заказы. Выберите другую.»

## Модель данных

Минимальный JSON для карточки (публичный API):

```json
{
  "slug": "baumana-10",
  "name": "Принт рядом",
  "address": "ул. Баумана, 10",
  "city": "kazan",
  "lat": 55.790,
  "lng": 49.120,
  "pricePerPageKopeks": 700,
  "openingHours": { "mon-fri": "9:00-19:00", "sat": "10:00-16:00" },
  "statusText": "Открыто до 19:00",
  "isOpenNow": true,
  "acceptsOnlineOrders": true,
  "agentOnline": true,
  "estimatedReadyMinutes": "1-3",
  "entryPhotoUrl": "https://...",
  "pickupInstructions": "После оплаты назовите оператору номер заказа",
  "services": [
    { "name": "Цветная печать", "priceFromKopeks": 2500 }
  ],
  "phone": "+7..."
}
```

`statusText`, `isOpenNow` — вычисляются на сервере из `openingHours` + timezone точки.
