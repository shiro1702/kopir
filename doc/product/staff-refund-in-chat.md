## Проблема

Сейчас возврат онлайн-оплаты (Т-Банк) доступен только в **веб-админке** (`/admin` → «Возврат» / «Возврат пачки`).

Сотрудник на точке работает в **Telegram / MAX** — там же приходят уведомления об оплате, сбоях печати и обращениях клиента. Если принтер замял бумагу, клиент недоволен качеством или передумал — сотруднику неудобно:

1. Открывать админку на телефоне
2. Искать заказ среди вкладок
3. Делать возврат отдельно от контекста переписки

Нужен возврат **в том же чате**, где сотрудник уже видит заказ.

---

## Идея

Добавить inline-кнопку **«💸 Вернуть оплату»** в staff-уведомления бота — рядом с уже существующими `staff_pay`, `staff_retry_print`, `staff_manual_print`.

Сотрудник нажимает → подтверждение → вызов того же API возврата, что и в админке (`refundTbankPaymentForOrder` / `refundTbankPaymentForBatch`) → клиенту сообщение в боте.

---

## Когда показывать кнопку

| Ситуация | Кнопка | Примечание |
|----------|--------|------------|
| Сбой печати (`FAILED`, автоповтор исчерпан) | ✅ | Рядом с «Повторить» / «Печать готова» |
| Клиент нажал «Что-то пошло не так» (Sprint 5+) | ✅ | После эскалации или сразу, если оплата онлайн |
| Уведомление «Оплата принята» | ❌ | Слишком рано — печать ещё не завершена |
| Оплата переводом / на месте | ❌ | Возврат вне Kopir (наличные, СБП на счёт копицентра) |
| Пачка (batch) | ✅ одна кнопка на пачку | «Вернуть оплату пачки» — один платёж на весь batch |

Условия как в админке:

- `paymentMethod` = `TBANK_SBP` или `TBANK_ONLINE`
- есть подтверждённый `Payment` с `externalId` (платёж пачки — на `batchId`)
- возврат ещё не выполнен (`Payment.status !== REFUNDED`)

## UX (черновик)

### Staff-чат

```
❌ Сбой печати
📦 Заказ #abc123 · 15 стр · 150 ₽
📍 ВСГУТУ, корпус 2
👤 @student_vasya
Ошибка: замятие бумаги

[🔄 Попробовать снова]  [✅ Печать готова]
[💸 Вернуть оплату (150 ₽)]
```

Нажатие «Вернуть оплату»:

```
Вернуть 150 ₽ клиенту через Т-Банк?
Заказ #abc123

[✅ Да, вернуть]  [Отмена]
```

После успеха — редактирование исходного сообщения:

```
💸 Возврат выполнен · 150 ₽
Заказ #abc123
```

### Клиентский чат

```
Мы вернули 150 ₽ на карту / счёт, с которого была оплата.
Обычно зачисление занимает до 3 рабочих дней.

Если нужна печать — отправьте файл снова.
```

## Техническая реализация (когда дойдём)

Переиспользовать уже существующий бэкенд:

```
web/server/utils/payments/providers/tbank-acquiring.ts
  → refundTbankPaymentForOrder / refundTbankPaymentForBatch
```

Новое:

| Компонент | Что сделать |
|-----------|-------------|
| `staff-notify.ts` | Добавить кнопку в клавиатуру сбоя / поддержки |
| `staff-actions.ts` | Callback `staff_refund:{orderId}` и `staff_refund_batch:{batchId}` |
| `staff-auth.ts` | Только привязанный staff точки (как у `staff_pay`) |
| `bot/core.ts` | Уведомление клиенту после возврата |
| Админка | Без изменений — дублирует тот же API |

Опционально: audit-log (`OrderEvent` / `StaffAction`) — кто и когда инициировал возврат.

---

## Ограничения и риски

- **Только онлайн-эквайринг** — ручные способы не трогаем.
- **Права** — callback принимает только staff, привязанный к `pointId` заказа (как подтверждение оплаты).
- **Двойной возврат** — кнопка скрывается после `REFUNDED`; повторный callback → «Уже возвращено».
- **Частичный возврат** — не в scope; только полная сумма платежа.
- **ФЗ-54 / чеки** — уточнить при prod (связано с PAY-04).

## Критерии готовности (когда возьмём в спринт)

- [ ] Кнопка «Вернуть оплату» в staff-уведомлении о сбое печати
- [ ] Подтверждение одним нажатием (inline confirm)
- [ ] Возврат через Т-Банк Cancel для одиночного заказа и пачки
- [ ] Клиент получает сообщение о возврате
- [ ] Повторный возврат невозможен
- [ ] E2E: sandbox-платёж → сбой → staff refund → статус `REFUNDED` в БД
