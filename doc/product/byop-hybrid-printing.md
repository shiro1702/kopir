# BYOP и гибридная печать (своя / наша бумага)

> Источник: [brainstorm/13.06.2026.md](../brainstorm/13.06.2026.md)  
> **Не в Sprint 0.** Этап боксов (Orange Pi + CUPS).

## Концепция BYOP

**BYOP** (Bring Your Own Paper) — пользователь вставляет свою бумагу в ручной слот (Manual Feed / Bypass Tray) перед печатью.

**Зачем:** нулевые расходы на бумагу, минимальное обслуживание, идеально для общежитий (у студента есть пачка, нет принтера).

| Параметр | Обычный LITE | LITE-BYOP |
|----------|--------------|-----------|
| Себестоимость | ~45 000 ₽ | **~28 000 ₽** |
| Бумага | наша, в нижнем лотке | **своя**, ручной слот |
| Замки / датчики бумаги | да | **не нужны** |
| Цена страницы (ориентир) | 10–15 ₽ | **4–5 ₽** (только тонер + амортизация) |

### Конструкция LITE-BYOP

- Корпус облегает принтер; нижний лоток **не используется** (или пустой).
- На лицевой панели — **наклонная прорезь** точно в Manual Feed.
- Выдача — верхний лоток, как обычно.

### Сценарий

1. Студент загружает PDF в бот.
2. Бот: «К печати N стр. Стоимость X ₽ (своя бумага). Вставьте N листов А4 в слот и нажмите „Оплатить“».
3. Оплата СБП → печать → забор из верхнего лотка.

## Гибридный режим (рекомендуется вместо «чистого» BYOP)

В боте пользователь выбирает тариф:

| Тариф | Источник бумаги | Цена (ориентир) | Аудитория |
|-------|-----------------|-----------------|-----------|
| **Всё включено** | Tray 1 (наш запас, замок) | ~10 ₽/стр | «быстро, лень искать бумагу» |
| **Своя бумага** | Manual Feed (щель на фасаде) | ~5 ₽/стр | экономные, со своей пачкой |

Принтер **не путает лотки** — источник задаётся в команде печати.

### CUPS / `lp`

```bash
# Своя бумага — ручной слот
lp -d Printer_Name -o InputSlot=Manual document.pdf

# Бумага принтера — нижний лоток
lp -d Printer_Name -o InputSlot=Tray1 document.pdf
```

Даже если в Manual Feed что-то торчит, при `InputSlot=Tray1` принтер берёт только из нижнего лотка.

### Конструкция гибридного LITE

```
┌─────────────────────────┐
│  Лоток выдачи           │
├─────────────────────────┤
│  Принтер + Orange Pi    │  ← LED, QR, экран PIN
│  [щель Manual Feed]     │  ← «Вставлять свою бумагу сюда» + LED
├─────────────────────────┤
│  Tray 1 (закрыт)       │  ← только для заправщика, электрозамок
└─────────────────────────┘
```

### UX при нехватке своей бумаги

Если студент оплатил 5 стр., вставил 3 листа: принтер останавливается (`Add Paper`). Orange Pi читает статус → бот: «Добавьте ещё 2 листа в слот».

### Режим «только своя бумага»

Если внутренний запас закончился: бот отключает «Наша бумага», предлагает BYOP со скидкой — бокс продолжает работать.

## Риски и митигация

| Риск | Решение |
|------|---------|
| Мятая / мокрая бумага → замятие | Инструкция на корпусе + предупреждение в боте; **без возврата** по вине пользователя |
| Картон, фольга, наклейки | Инфографика: «Только А4»; Pantum/Brother — живучие, но наклейки опасны для вала |
| Меньше листов, чем страниц | Статус `Add Paper` + push в бот |
