Обзор: путь клиента

Упрощённая схема сверху вниз. Цвет рамки = статус реализации.

flowchart TB
  START["/start ✅"] --> BATCH["Сбор файлов\nдо 5 · Отменить всё"]
  BATCH --> PAY_BTN["«Оплатить» ✅"]
  BATCH --> CANCEL["Сброшено ✅"]
  PAY_BTN --> METHODS{"Способ оплаты"}
  METHODS -->|перевод| TR["СБП + Я оплатил ✅"]
  METHODS -->|на месте| OS["Стойка ✅"]
  METHODS -->|СБП| TB1["Т-Банк СБП ✅"]
  METHODS -->|карта| TB2["pay.tbank.ru ✅"]
  TR --> STAFF["Staff confirm ✅"]
  OS --> STAFF
  TB1 --> PAID["PAID webhook ✅"]
  TB2 --> PAID
  STAFF --> PAID
  PAID --> PRINT["Печать ✅"]
  PRINT --> DONE["Готово! 🟡"]
  PRINT --> RETRY["Retry ✅"]

Диаграмма состояний

Полный state machine клиента с вложенными состояниями пачки и оплаты.

stateDiagram-v2
    direction TB
  [*] --> Idle: /start ✅
  Idle --> Collecting: PDF/Word ✅
  state Collecting {
    [*] --> FileReceiving
    FileReceiving --> FileReady: PDF ✅
    FileReceiving --> FileCalculating: Word ✅
    FileCalculating --> FileReady: посчитано ✅
    FileReady --> FileRemoved: удалить ✅
  }
  Collecting --> AwaitingPayment: Оплатить ✅
  Collecting --> Cancelled: Отменить всё ✅
  state AwaitingPayment {
    [*] --> ChooseMethod
    ChooseMethod --> Transfer: перевод ✅
    ChooseMethod --> OnSite: на месте ✅
    ChooseMethod --> OnlineSbp: СБП ✅
    ChooseMethod --> OnlineCard: карта ✅
    Transfer --> StaffConfirm: Я оплатил ✅
    OnSite --> StaffConfirm: staff ✅
    OnlineSbp --> Paid: webhook ✅
    OnlineCard --> Paid: webhook ✅
  }
  StaffConfirm --> Paid: staff confirm ✅
  Paid --> Printing: print ✅
  state Printing {
    [*] --> InQueue
    InQueue --> Done: успех ✅
    InQueue --> Failed: ошибка ✅
    Failed --> InQueue: retry ✅
  }
  Done --> [*]: Готово 🟡

Онлайн-оплата: Telegram vs MAX

Полная спека: online-payment-bot-flow.md

sequenceDiagram
  participant C as Клиент
  participant TG as Telegram
  participant B as Бот
  participant T as Т-Банк

  Note over C,T: Telegram СБП/карта
  C->>B: тап СБП или карта callback
  B->>B: selectPaymentMethod + initPayment
  B->>TG: answerCallbackQuery url=pay.tbank.ru
  TG->>T: открывает банк
  B->>C: Подтвердите + Проверить
  T->>B: webhook CONFIRMED
  B->>C: Оплата получена

  Note over C,T: MAX СБП/карта
  C->>B: Оплатить
  B->>B: prepareMaxOnlinePaymentUrls
  B->>C: кнопки СБП/карта = прямые url
  C->>T: тап ссылки
  T->>B: webhook CONFIRMED
  B->>C: Оплата получена

  B->>A: print

  B->>B: Init + open TG / url MAX
  T->>B: webhook CONFIRMED
  B->>A: print без staff

Архитектура каналов

Telegram и MAX → общее ядро bot/core.ts.

flowchart LR
  U[Пользователь] --> TG[Telegram]
  U --> MAX[MAX]
  TG --> CORE[bot/core.ts]
  MAX --> CORE
  CORE --> PH[payment-handlers]
  CORE --> CB[client-callbacks]
  PH --> ST[Staff чат]
  CORE --> ST
  CORE --> API[Nitro API]
  API --> AGT[Desktop агент]
