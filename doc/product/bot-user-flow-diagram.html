Обзор: путь клиента

Упрощённая схема сверху вниз. Цвет рамки = статус реализации.

flowchart TB
  START["/start ✅"] --> BATCH["Сбор пачки\nPDF · Word · 🗑"]
  BATCH --> PAY_BTN["«Оплатить» ✅"]
  BATCH --> CANCEL["Отмена ✅"]
  PAY_BTN --> METHODS{"Способ оплаты"}
  METHODS -->|перевод| TR["СБП + Я оплатил ✅"]
  METHODS -->|на месте| OS["Стойка ✅"]
  METHODS -->|онлайн| TB["Т-Банк 🟡"]
  TR --> STAFF["Staff confirm ✅"]
  OS --> STAFF
  TB --> PAID["PAID ✅"]
  STAFF --> PAID
  PAID --> PRINT["Печать ✅"]
  PRINT --> DONE["Готово! 🟡"]
  PRINT --> RETRY["Retry ✅"]

Диаграмма состояний

Полный state machine клиента с вложенными состояниями пачки и оплаты.

stateDiagram-v2
    direction TB
  [*] --> Idle: /start ✅
  Idle --> Collecting: PDF/Word ✅
  state Collecting {
    [*] --> FileReceiving
    FileReceiving --> FileReady: PDF ✅
    FileReceiving --> FileCalculating: Word ✅
    FileCalculating --> FileReady: посчитано ✅
    FileReady --> FileRemoved: удалить ✅
  }
  Collecting --> AwaitingPayment: Оплатить ✅
  Collecting --> Cancelled: Отменить ✅
  state AwaitingPayment {
    [*] --> ChooseMethod
    ChooseMethod --> Transfer: перевод ✅
    ChooseMethod --> OnSite: на месте ✅
    ChooseMethod --> Online: Т-Банк 🟡
    Transfer --> StaffConfirm: Я оплатил ✅
    OnSite --> StaffConfirm: staff ✅
    Online --> Paid: webhook 🟡
  }
  StaffConfirm --> Paid: staff confirm ✅
  Paid --> Printing: print ✅
  state Printing {
    [*] --> InQueue
    InQueue --> Done: успех ✅
    InQueue --> Failed: ошибка ✅
    Failed --> InQueue: retry ✅
  }
  Done --> [*]: Готово 🟡

Сбор файла

Условия показа кнопок «Оплатить» и «Удалить».

flowchart TD
  A[Документ] --> B{Тип?}
  B -->|PDF| C[AWAITING_PAYMENT]
  B -->|Word| D[CALCULATING]
  B -->|другое| E[Отказ]
  C --> F["Принимаю → edit"]
  D --> F
  F --> G{Blob OK?}
  G -->|нет| H[ошибка]
  G -->|PDF| I["Файл готов + 🗑"]
  G -->|Word| J["Считаем…"]
  J --> K{Агент}
  K -->|ok| I
  K -->|fail| L["❌ + 🗑"]
  I --> M{keyboardMode}
  M -->|ready| N["Оплатить + Отменить"]
  M -->|calculating| O["Только Отменить"]

Оплата

Три ветки после finalizeBatch. Фильтр методов — payment-config.ts.

flowchart TD
  F1[Сводка пачки] --> F2[Выбор способа]
  F2 --> M1[Перевод ✅]
  F2 --> M2[На месте ✅]
  F2 --> M3[Онлайн 🟡]
  M1 --> T1[Реквизиты + Я оплатил]
  M2 --> T2[Инструкция стойка]
  M3 --> T3[QR + Проверить]
  T1 --> ST1[Staff: банк]
  T2 --> ST2[Staff: стойка]
  T3 --> PAID[PAID webhook]
  ST1 --> CONF[staff confirm]
  ST2 --> CONF
  CONF --> PAID
  PAID --> PRINT[Агент печатает]

Архитектура каналов

Telegram и MAX → общее ядро bot/core.ts.

flowchart LR
  U[Пользователь] --> TG[Telegram]
  U --> MAX[MAX]
  TG --> CORE[bot/core.ts]
  MAX --> CORE
  CORE --> PH[payment-handlers]
  CORE --> CB[client-callbacks]
  PH --> ST[Staff чат]
  CORE --> ST
  CORE --> API[Nitro API]
  API --> AGT[Desktop агент]
