# Мультигород, deep links, один бот

Источник: [брейншторм 28.06.2026](../brainstorm/28.06.2026.md)

**Детальный UX выбора точки в боте:** [bot-point-selection.md](./bot-point-selection.md) (брейншторм 30.06).

Мультигород закладываем **с первого дня**, даже при старте только в Улан-Удэ.

## База данных

```
Cities ──< Points ──< Orders
```

Аналитика по городам: «Казань 10k стр/мес, Н. Новгород 3k».

---

## Клиент (B2C)

### Сайт

- Селектор города в header (как Авито / Додо)
- Первый визит: IP → город, иначе выбор из списка
- Карта центрируется на выбранном городе

### Telegram / Mini App

**Приоритет 1 — QR на точке (95% сценариев):**

```
t.me/kopir_bot?start=point_105
```

Бот читает `point_105` → город + точка автоматически. Вопрос про город **не задаётся**.

**Приоритет 2 — заход без QR:**

1. Кнопка «Поделиться геопозицией»
2. Если отказ — ручной выбор города (fallback)

### Партнёр при создании точки

Обязательные поля:
- Название
- **Город** (dropdown; «Запросить новый город» — сигнал для экспансии)
- Адрес
- Координаты (клик на карте)
- Принтер + токен

---

## Один бот vs бот на город

**Решение: один федеральный бот.**

| Критерий | Один бот | Бот на город |
|----------|----------|--------------|
| Поездка в другой город | QR → та же история, карта | Новый бот, заново |
| Маркетинг | Одна ссылка | N ссылок |
| Обновления | 1 деплой | N токенов |
| Лимиты TG | 30 msg/s — запас на сотни точек | — |

Партнёры могут давать **свою** ссылку с `?start=point_XXX` — это тот же бот.

---

## UX-правило

> 95% случаев: QR или гео → город и точка без ручного ввода.  
> Ручной выбор — только резерв.

При регистрации точки партнёром: **Улан-Удэ по гео/IP по умолчанию**, можно поправить.
