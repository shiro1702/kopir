# Конвертация DOC/DOCX

> Источник: [brainstorm/12.06.2026.md](../brainstorm/12.06.2026.md)  
> **Sprint 0:** только PDF. DOCX — Sprint 1+.

## Проблема

Один и тот же `.docx` может дать разное число страниц на сервере и на принтере копицентра. Клиент должен платить за то, что реально выйдет из принтера.

## Три подхода

### 1. Локальный MS Word (MVP для копицентров)

- Python + `pywin32` на Windows ПК партнёра
- Подсчёт страниц и печать в фоне
- **Плюсы:** точность 100%, 0 нагрузки на Vercel, быстрый старт
- **Минусы:** нужен Word, риск popup'ов, макросы, очередь при пике

### 2. LibreOffice headless на ПК

- Бесплатно, но искажения сложных таблиц
- Всё равно зависимость от ПК партнёра

### 3. Серверная конвертация (целевое для боксов)

- Gotenberg / Loxia на VPS (~700 ₽/мес)
- Vercel пересылает файл → PDF в Blob → агент печатает PDF
- **Обязательно** для Orange Pi / Linux боксов (нет Word)

## Схема «двойного рукопожатия» (MVP)

```
Пользователь → загрузка .docx
Сервер → order status: pending_calculation
Агент → считает страницы в Word → POST page_count
Бот → цена → оплата
Сервер → status: paid
Агент → печать
```

Задержка 5–10 сек — норма для MVP («Бот считает страницы…»).

## Почему не качаем из Telegram в Python

1. **Безопасность** — токен бота не должен быть на чужом ПК
2. **Ссылки Telegram** живут ~1 час
3. **Конвертация** — принтеру нужен стандартизированный PDF
4. **Блокировки** — в вузах могут резать `*.telegram.org`
