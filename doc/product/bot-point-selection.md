## Два сценария входа

| Сценарий | Порядок | Когда |
|----------|---------|-------|
| **У принтера** | Точка → файлы → оплата | QR на стойке, код точки |
| **Из дома / в пути** | Файлы → точка → оплата | Зашёл в бота без QR |

Бот хранит в сессии **два параметра**: список файлов (пачка) и выбранная точка. Сообщение-карточка заказа обновляется через `editMessageText`.

Бот **не может** сам написать пользователю при повторном заходе — только реагировать на действие (файл, кнопка, `/start`). Persistent Menu: `/print`, `/help`.

## Способы выбора точки

| Способ | Приоритет MVP | Статус |
|--------|---------------|--------|
| QR / deep link `?start=point_<slug>` | P0 | ✅ Sprint 0 |
| Текстовый код точки (`102` или `/start 102`) | P1 | ⬜ UX-06 |
| Последняя точка (автовыбор) | P0 | ⬜ |
| Плоский список точек города | P1 | ⬜ |
| Геолокация «Найти ближайший» | P2 | ⬜ UX-07 |
| Mini App / карта | P3 (последняя очередь) | ⬜ WEB-10 |
| Иерархия Город → ВУЗ → Корпус | Отложено | ⏸ при масштабе |
| Inline-поиск `@bot всгуту` | Nice-to-have | ⏸ |

### Рекомендуемое меню «Изменить точку»

Показывать **только** когда пользователь явно меняет точку (не при автовыборе последней):

1. **↩️ Вернуть: [последняя точка]** — только если сейчас выбрана другая, а в истории есть предыдущая
2. **📋 Выбрать из списка** — плоский список точек **города клиента** (без иерархии на старте)
3. **📍 Найти ближайшую** — запрос геопозиции
4. **🗺 Выбрать на карте** — Mini App (реализовать в последнюю очередь)
5. **⬅️ Назад к оплате**

Подсказка в тексте: *«Или отправьте номер точки, если вы уже у принтера»*.

## Автовыбор последней точки

### Данные в БД (на пользователя)

| Поле | Назначение |
|------|------------|
| `last_point_id` | Последняя точка, где печатал |
| `user_city` | Город по последней точке; для нового — дефолт (Улан-Удэ) |

### Поведение

**Повторный клиент** — при приёме файла точка подставляется автоматически, кнопка «вернуть последнюю» **не показывается**:

```
📥 Файл принят: diplom_final.pdf (15 стр.)
Вы можете отправить еще файлы, они добавятся в эту же пачку.

📍 Точка: ВСГУТУ, Корпус 2 (код 102) — выбрана автоматически
💰 Стоимость: 75 ₽

[💳 Перейти к оплате (75 ₽)]
[📍 Изменить точку]
[❌ Удалить файл]
```

**Новый клиент или смена точки** — точка не выбрана до явного действия:

```
📥 Файл принят: diplom_final.pdf (15 стр.)
Вы можете отправить еще файлы, они добавятся в эту же пачку.

⚠️ Чтобы рассчитать стоимость и отправить на печать, выберите точку:

[📋 Выбрать из списка]
[📍 Найти ближайшую]
[❌ Удалить файл]
```

*(Кнопка карты — когда Mini App готов.)*

## Карточка заказа (динамическая)

### Точка не выбрана

```
📄 Ваш заказ:
1. diplom_v1.pdf (40 стр., ч/б)

⚠️ Для расчета стоимости и оплаты выберите принтер.

[📍 Выбрать принтер]
[❌ Удалить файл]
```

### Точка выбрана (QR или после выбора)

```
📄 Ваш заказ:
1. diplom_v1.pdf (40 стр., ч/б)

📍 Точка: ВСГУТУ, Корпус 2 (код 102)
💰 Итого: 200 ₽

[💳 Оплатить и распечатать]
[📍 Изменить точку]
[❌ Удалить файл]
```

## Пачка файлов: что НЕ делаем

| Решение | Причина |
|---------|---------|
| ❌ Кнопка «Добавить файл» | Пользователь просто отправляет следующий файл в чат |
| ❌ Загрузка файлов в Mini App (сейчас) | Mini App только для карты и выбора точки |
| ❌ Инструкция iOS «скрепка → Файл» | Лишний шум; одна строка «можно отправить ещё файлы» достаточно |

При втором файле — **edit** того же сообщения, список растёт.

## Сценарий A: QR на точке

1. `t.me/kopir_bot?start=point_102` → точка сразу в сессии
2. Приветствие: *«📍 Выбрана точка: … Отправьте файлы»*
3. Файлы → карточка с ценой и «Оплатить»
4. Если точка offline — см. [point-availability.md](./point-availability.md)

---

## Сценарий B: файлы без точки

1. Пользователь кидает файл → карточка без цены, «Выбрать принтер»
2. Выбор точки (список / гео / код / карта)
3. Карточка обновляется с ценой и «Оплатить»
