## Этап 1 — Ручной пилот (Proof of Concept) ✅ по коду

**Цель:** доказать, что файл из мессенджера доезжает до принтера за < 30 сек.

**Бизнес:**
- 1 дружественный копицентр (или домашний принтер)
- Тест только на себе и друзьях
- Оплата вручную (перевод / на месте → подтверждение staff)

**Техника (сделано):**
- Nuxt: боты Telegram + MAX на общем ядре `bot/core`
- Python: polling → печать PDF и DOCX (Word на Windows)
- `OrderBatch`: несколько файлов, одна оплата
- Выбор способа оплаты (перевод СБП / на месте), слой `PaymentProvider`
- Админка заказов `/admin` (пачки + одиночные)
- Без эквайринга, без WebSocket, без GUI

**Спринты:** Sprint 0, 0.1, 0.2

**Критерий готовности:**
- [x] Код: бот → API → агент → печать (PDF + DOCX)
- [x] Код: batch + ручная оплата + staff-уведомления
- [ ] E2E: 5+ batch-заказов подряд без регрессии (Sprint 0.2, задача 07)
- [ ] E2E docx на Windows (Sprint 0.1)
- [ ] Договорённость с 1 копицентром на бета

## Этап 2a — Бета в 1 копицентре (Sprint 1)

**Цель:** 20–30 реальных пользователей на одной точке, стабильный агент, ручная оплата.

**Бизнес:**
- 1 копицентр (пилот БГУ / ВСГУТУ)
- Оплата: перевод СБП или на месте (без эквайринга)
- QR на стойке → бот с привязкой к точке
- Сбор feedback и метрик

**Техника (план / в работе):**
- `kopir-agent.exe` + установка на ПК партнёра
- WebSocket hub (VPS) + heartbeat online/offline
- Блокировка новых заказов на offline-точку
- Разделительный лист (toggle)
- QR deep link + PDF-плакат
- Уведомление «Готово!» (TG + MAX)
- Демо-оплата (опционально, только UX-тесты; основной flow — ручной)

**Уже сделано вне графика Sprint 1:**
- Выбор способа оплаты (PAY-01b)
- Удаление файла из пачки, статусные сообщения бота
- Ручная печать при сбое агента
- Heartbeat MVP: `lastSeenAt` + индикатор в админке (без блокировки до оплаты)

**Критерий готовности:**
- [ ] Агент установлен в копицентре, WS end-to-end
- [ ] 30+ заказов за спринт, < 5% ошибок печати
- [ ] QR на стойке, конверсия зафиксирована

**Спринт:** Sprint 1 (26 июня – 9 июля 2026)

## Этап 2b — Коммерческий MVP (Sprint 2)

**Цель:** реальные деньги через Т-Банк, 2–3 точки, минимальное управление точками без полного онбординга.

**Бизнес:**
- 2–3 партнёра (копицентры у БГУ / ВСГУТУ)
- Модель 80/20, бесплатный тест 2 недели
- Выплаты вручную раз в неделю (СБП)
- Агентский договор-оферта на сайте

**Техника — приоритет (порядок разработки):**

| # | Блок | Зачем |
|---|------|-------|
| 1 | **Т-Банк СБП** (Init, GetQr, webhook на `OrderBatch`) | Онлайн-оплата без staff |
| 2 | **Провайдер `tbank`** в существующий `PaymentProvider` | Тот же flow статусов, что ручная оплата |
| 3 | **Минимальная админка точек** | CRUD 2–3 точек: slug, цена, реквизиты, способы оплаты, online/offline |
| 4 | **`/bind <token>`** для staff-канала на точку | Масштаб без redeploy env |
| 5 | Облачная касса (тест) | ФЗ-54 после первых транзакций |
| 6 | Mini App, калькулятор, превью | Параллельно или после первых платежей |
| 7 | GUI агента PySide6 | Удобство партнёра, не блокер оплаты |

**Минимальная админка точек (MVP, не полный кабинет партнёра):**
- Список точек: название, slug, `transferPhone`, цена за страницу
- Переключение способов оплаты: `SBP_TRANSFER` / `ON_SITE` / `TBANK_ONLINE`
- Генерация одноразового bind-токена для staff
- Статус агента (online/offline)

> Полный self-service онбординг и partner dashboard → **Этап 3** (Sprint 3).

**Критерий готовности:**
- [ ] 100+ реальных транзакций через Т-Банк за 2 недели
- [ ] > 95% успешных печатей
- [ ] 3 активные точки, staff привязан через `/bind`
- [ ] Первая выплата партнёру

**Спринт:** Sprint 2 (10–23 июля 2026, сдвиг +1 нед после закрытия Sprint 1)

## Этап 3 — Полная автоматизация (Scale-Ready)

**Цель:** основатель не участвует в подключении точек и расчётах.

**Бизнес:**
- Саморегистрация партнёров
- Автовыплаты (Mass Payments)
- Публичная оферта
- Первые 1–3 собственных бокса (опционально)

**Техника:**
- Онбординг: токен, авто-привязка принтера
- **Partner dashboard v1** (статистика, баланс, «Вывести») — WEB-13
- Статичная карта точек (Leaflet)
- Redis heartbeat
- Deploy staging + prod VPS
- Превью страниц, статус печати в реальном времени

**Критерий готовности:**
- [ ] Новая точка подключается без созвона (< 15 мин)
- [ ] Выплаты без ручного участия (пилот Mass Payments)
- [ ] Uptime точек > 99% в рабочие часы

**Спринты:** Sprint 3–4

## Этап 4 — Масштабирование

**Цель:** захват Улан-Удэ, выход в Иркутск / Чита.

**Бизнес:**
- 20+ точек в городе
- Маркетинг в студенческих пабликах
- Франшизный пакет
- B2B SaaS для сетей копицентров

**Техника:**
- **Super-admin** (GMV, комиссии) — WEB-12
- VK-бот, мультиссылка QR
- Карта с нагрузкой в реальном времени
- Мониторинг бумаги/тонера, алерты
- Windows Service для агента
- Собственные принт-боксы (Linux agent)

**Критерий готовности:**
- [ ] 5 000+ активных пользователей
- [ ] Положительная unit-экономика на точку
- [ ] Второй город запущен

**Спринты:** Sprint 5+
