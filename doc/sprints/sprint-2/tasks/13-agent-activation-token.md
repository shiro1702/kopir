## Проблема

Сейчас агент привязан к точке через `POINT_ID` в `.env` — при добавлении 2–3 точки нужно вручную править конфиг на каждом ПК.

## Цель

Админ генерирует одноразовый токен для точки → партнёр вводит его в агенте (или в `.env` `ACTIVATION_TOKEN=...`) → агент получает `pointId` / `slug` с сервера и сохраняет локально.

**Не полный онбординг** (регистрация партнёра на сайте) — только привязка агента к существующей точке. Self-service → Sprint 4.
