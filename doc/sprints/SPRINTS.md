# Kopir — спринты

**Текущая дата:** 28 июня 2026  
**Активный спринт:** [Sprint 2 — Подготовка к коммерции](./sprint-2/README.md) (админка точек, `/bind`, скелет Т-Банка)  
**Заблокирован:** [Sprint 3 — Т-Банк](./sprint-3/README.md) (ждём номинальный счёт в банке)  
**Горизонт:** запуск к 1 сентября 2026

> Продуктовые этапы: [roadmap/ROADMAP.md](../roadmap/ROADMAP.md)

## Карта спринтов (пересборка 28.06.2026)

```
Sprint 0–0.2   Пилот (код ✅)
Sprint 1       Бета в 1 копицентре, polling (без WebSocket) — закрывается
Sprint 2  ◄──  АКТИВНЫЙ: масштаб на ручной оплате + подготовка эквайринга
Sprint 3       Т-Банк + касса — старт после одобрения банка
Sprint 4       Онбординг партнёра без созвона
Sprint 5       К 1 сентября (стабильность, 5+ точек)
Sprint 6       Рост
Backlog        WebSocket, VPS hub, Redis — когда понадобится масштаб
```

**Ключевые решения:**
- WebSocket **не делаем** в Sprint 1 — polling + `lastSeenAt` достаточно для 1–3 точек
- Пока ждём банк — **Sprint 2**, не полный Sprint 4 (онбординг)
- Т-Банк выделен в отдельный **Sprint 3** (короткий, после ключей)

---

## Как пользоваться

- Спринт = **1–2 недели** (Sprint 3 может быть короче — только интеграция)
- Задачи ссылаются на ID из [FEATURES.md](../project/FEATURES.md)
- Статусы: `⬜` / `🔵` / `✅` / `❌ cancelled` / `⏸ blocked`
- В конце спринта — ретро по шаблону ниже
