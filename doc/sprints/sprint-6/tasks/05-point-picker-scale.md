## Цель

MVP **этап 3**: когда точек > 10 — фильтры, поиск, кластеризация маркеров, форма «нет точки рядом», базовая аналитика.

## Scope

### Фильтры и сортировка

- [ ] **5.1** UI фильтров: «Открыто сейчас», «Рядом», «Дешевле», «Быстрее»
- [ ] **5.2** Фильтры по услугам (если `Point.services` заполнены)
- [ ] **5.3** Dropdown сортировки: расстояние / цена / открытые первыми
- [ ] **5.4** Client-side filter на данных `usePointPicker` (без нового API на старте)

### Поиск

- [ ] **5.5** Поле «Адрес, район, метро, название»
- [ ] **5.6** MVP: fuzzy match по `name`, `address`, `city` (без геокодера)
- [ ] **5.7** P2: DaData geocode → центр карты на район

### Кластеризация

- [ ] **5.8** `leaflet.markercluster` при > 15 маркерах в viewport
- [ ] **5.9** Кластер показывает count, zoom → раскрытие

### Пустое состояние и lead-форма

- [ ] **5.10** «Нет точек рядом» — [Все точки] [Предложить район]
- [ ] **5.11** Форма: город, район, комментарий (вуз/БЦ) → `POST /api/leads/point-request` или email
- [ ] **5.12** Сообщение пилота: «Сервис расширяется»

### SEO интеграция

- [ ] **5.13** «Подробнее» на карте → `/print/[city]/[slug]` ([Sprint 5 / 06](../../sprint-5/tasks/06-print-point-pages.md))
- [ ] **5.14** JSON-LD `LocalBusiness` на странице точки

### Аналитика (P2)

- [ ] **5.15** Events: `point_view`, `point_select`, `point_route_click`, `geo_request`
- [ ] **5.16** Яндекс.Метрика goals (если подключена в S5 / 09)

## Не в scope

- Динамическая загрузка по очереди Spooler (WEB-11 full) → backlog
- Сравнение точек таблицей → nice-to-have

## DoD

- [ ] 20+ точек на карте: кластеры работают
- [ ] Фильтр «Открыто сейчас» скрывает закрытые
- [ ] Поиск по названию/адресу находит точку
- [ ] Форма «нужна точка» сохраняет lead
