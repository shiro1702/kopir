# 12 — Ручная печать при сбое агента

| | |
|---|---|
| **Статус** | ✅ done |
| **Feature** | MON-09 |
| **Зависимости** | Sprint 0: staff-notify, Blob; задача 09 (уведомления при FAILED) |
| **Оценка** | 3–4 часа |

## Цель

Если автоматическая печать на ПК агента упала, но оплата уже принята, сотрудник копицентра получает **файл заказа** в staff-канал (MAX / Telegram) и после ручной печати нажимает **«Печать готова»**. Клиент получает «Готово!», файл удаляется из Blob.

## Сценарий

```
Агент: download/print FAILED
    → Order.status = FAILED, errorMessage в БД
    → Клиент: «Не удалось напечатать… обратитесь к сотруднику»
    → Staff: текст + файл + кнопка «✅ Печать готова»
    → Сотрудник печатает вручную (сохранённый PDF/DOCX из бота)
    → Staff нажимает «Печать готова»
    → Order.status = PRINTED, клиент «Готово!», blob удалён
```

## Подзадачи

- [x] **12.1** При `FAILED` от агента **не удалять** файл из Blob (удаление только после `PRINTED` или ручного подтверждения)
- [x] **12.2** `notifyStaffPrintFailed`: переслать файл в MAX (`POST /uploads` → `POST /messages`) и Telegram (`sendDocument`)
- [x] **12.3** Кнопка staff `staff_manual_print:{orderId}` — «✅ Печать готова»
- [x] **12.4** `confirmManualPrint(orderId)` — `FAILED` → `PRINTED`, уведомить клиента, удалить Blob; для пачки — `checkBatchCompletion`
- [x] **12.5** Тексты: staff видит причину сбоя + инструкцию; клиент не видит технических деталей

## Критерии приёмки

- [x] Сбой агента → staff получает файл в MAX (если `STAFF_MAX_USER_ID` задан)
- [x] После «Печать готова» клиент получает «Готово!»
- [x] Повторное нажатие на уже `PRINTED` — без ошибки (идемпотентно)
- [x] Одиночный заказ и файл внутри пачки работают одинаково

## Заметки

- Только staff-каналы из env (`STAFF_TELEGRAM_CHAT_ID`, `STAFF_MAX_USER_ID`); `/bind` — Sprint 2
- Если файл не удаётся отправить (Blob недоступен) — staff всё равно получает текст с причиной; ручная печать через админку / повтор заказа
- Возврат денег (PAY-11) — отдельная фича, не в этом scope
