# 06 — Уведомления клиенту по пачке

| | |
|---|---|
| **Статус** | ⬜ todo |
| **Feature** | UX-10, UX-12 |
| **Зависимости** | 02, 03, 04 |
| **Оценка** | 2–3 часа |

## Цель

Сделать понятные сообщения пользователю на всех этапах batch-flow: собрали пачку, оплатили, начали печатать, завершили.

## Подзадачи

- [ ] **6.1** Сообщение «Пачка собрана» (файлы, страницы, сумма)
- [ ] **6.2** Сообщение «Оплата подтверждена» по `batchId`
- [ ] **6.3** Сообщение «Печать началась» (общий прогресс `1/N`)
- [ ] **6.4** Сообщение «Готово» по всей пачке
- [ ] **6.5** Ошибка печати в одном файле:
  - указать имя файла
  - дать понятную инструкцию (обратиться к сотруднику)

## Критерии приёмки

- [ ] Пользователь видит единый flow по пачке, без спама по каждому служебному изменению
- [ ] Сообщения работают в Telegram и MAX
- [ ] При частичной ошибке остаётся понятный финальный статус

## Заметки

- Прогресс можно слать только при старте и завершении, без каждого шага
- Формулировки сохраняем короткими и дружелюбными
