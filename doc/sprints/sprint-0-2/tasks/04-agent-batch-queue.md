# 04 — Agent queue: печать пачки по порядку

| | |
|---|---|
| **Статус** | ⬜ todo |

| **Зависимости** | 01, 03 |
| **Оценка** | 3–4 часа |

## Цель

После оплаты пачки агент должен печатать её файлы строго по порядку `batchIndex`.

## Подзадачи

- [ ] **4.1** Расширить очередь агента полями `batchId`, `batchIndex`
- [ ] **4.2** Сортировка выдачи:
  - сначала по времени создания пачки
  - внутри пачки по `batchIndex`
- [ ] **4.3** Логика завершения:
  - `complete` на каждом `Order`
  - финальный статус пачки, когда напечатан последний файл
- [ ] **4.4** Обработка ошибки:
  - зафиксировать ошибочный `Order`
  - продолжать/останавливать пачку по флагу (по умолчанию продолжать)
- [ ] **4.5** Обновить payload для desktop-агента

## Критерии приёмки

- [ ] Файлы в пачке печатаются в ожидаемой последовательности
- [ ] Параллельные пачки не перемешивают порядок внутри себя
- [ ] При падении на одном файле диагностический статус виден staff

## Заметки

- Разделительный лист между файлами пачки не обязателен в 0.2
- Важно не ломать старые одиночные заказы без `batchId`
