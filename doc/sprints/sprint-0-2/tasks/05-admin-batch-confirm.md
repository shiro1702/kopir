## Цель

В админке staff подтверждает оплату один раз на всю пачку, а не по каждому файлу.

## Подзадачи

- [ ] **5.1** Группировка таблицы заказов по `batchId`
- [ ] **5.2** Кнопка `Подтвердить оплату пачки`
- [ ] **5.3** Отображение в UI:
  - число файлов
  - общая сумма
  - прогресс печати в пачке
- [ ] **5.4** Текст ошибок для некорректного статуса пачки
- [ ] **5.5** Fallback: если заказ без `batchId`, работает старый action по одному заказу

## Критерии приёмки

- [ ] Staff может подтвердить пачку одной кнопкой
- [ ] В таблице понятно, какие файлы относятся к какой пачке
- [ ] Старый flow одиночного заказа не сломан

## Заметки

- Режим оплаты остаётся terminal/demo
- Онлайн-оплата появится в Sprint 2
