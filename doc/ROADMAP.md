# Kopir — Roadmap

```
Этап 1          Этап 2              Этап 3                 Этап 4
Ручной пилот →  MVP (3–5 точек) →  Автоматизация →        Масштабирование
1–2 недели      3–4 недели          1–2 месяца             3+ месяца
```

---

## Этап 1 — Ручной пилот (Proof of Concept)

**Цель:** доказать, что файл из Telegram доезжает до принтера за < 30 сек.

**Бизнес:**
- 1 дружественный копицентр (или домашний принтер)
- Тест только на себе и друзьях
- Оплата вручную (перевод → подтверждение в админке)

**Техника:**
- Nuxt: простой бот принимает PDF
- Python: polling каждые 5 сек → печать
- Без эквайринга, без GUI

**Критерий готовности:**
- [ ] 10+ успешных печатей подряд
- [ ] Время от загрузки до печати < 45 сек

**Спринт:** Sprint 0

---

## Этап 2 — MVP для первых 3–5 точек (Улан-Удэ)

**Цель:** коммерческая эксплуатация, первые деньги, бета в копицентре.

**Бизнес:**
- 3–5 партнёров (копицентры у БГУ / ВСГУТУ)
- Модель 80/20, бесплатный тест 2 недели
- Выплаты вручную раз в неделю (СБП)
- Агентский договор-оферта на сайте

**Техника:**
- Telegram Mini App: загрузка, превью, калькулятор
- Т-Банк СБП + webhook
- Python GUI (PySide6) + WebSocket
- Разделительный лист (toggle)
- QR-материалы из программы
- Partner dashboard v1 (таблица заказов)
- Heartbeat online/offline
- Статичная карта точек

**Критерий готовности:**
- [ ] 100+ реальных транзакций за 2 недели
- [ ] > 95% успешных печатей
- [ ] Сессия пользователя < 1 мин

**Спринты:** Sprint 1–2

---

## Этап 3 — Полная автоматизация (Scale-Ready)

**Цель:** основатель не участвует в подключении точек и расчётах.

**Бизнес:**
- Саморегистрация партнёров
- Автовыплаты (Mass Payments)
- Публичная оферта
- Первые 1–3 собственных бокса (опционально)

**Техника:**
- Онбординг: токен, авто-привязка принтера
- Partner + Super-admin дашборды v2
- Мультиссылка QR (TG / Viber / VK)
- Карта с нагрузкой в реальном времени
- Мониторинг бумаги/тонера, алерты
- Очередь печати, частичные возвраты
- Печать по QR на боксе (безопасность)
- Windows Service для агента
- Облачная касса в продакшене

**Критерий готовности:**
- [ ] Новая точка подключается без созвона (< 15 мин)
- [ ] Выплаты без ручного участия
- [ ] Uptime точек > 99% в рабочие часы

**Спринты:** Sprint 3–4

---

## Этап 4 — Масштабирование

**Цель:** захват Улан-Удэ, выход в Иркутск / Чита.

**Бизнес:**
- 20+ точек в городе
- Маркетинг в студенческих пабликах
- Франшизный пакет
- B2B SaaS для сетей копицентров

**Техника:**
- Боты Viber / VK на общем API
- Реферальная программа
- Динамическое ценообразование
- Прогноз расходников
- Собственные принт-боксы (Linux agent)
- Локальный Wi-Fi на боксах (опционально)

**Критерий готовности:**
- [ ] 5 000+ активных пользователей
- [ ] Положительный unit-экономика на точку
- [ ] Второй город запущен

**Спринты:** Sprint 5+

## Стратегия продуктовой линейки

```
Сейчас                    Через 6 мес               Через 12 мес
─────────────────────────────────────────────────────────────────
SaaS на ПК партнёра  →   + собственные боксы   →   + франшиза
(0 CapEx)                  (~35k ₽/точка)            (другие города)
```

**Не делаем на старте:** Orange Pi, металлические корпуса, POS-терминалы, МФЦ, термопринтеры.

## Приоритеты при конфликте

1. Надёжная печать PDF на Windows
2. Оплата СБП без сбоев
3. UX < 1 минуты
4. Партнёрский дашборд
5. Всё остальное
